# 離席・中断の検知

## 解こうとしている問題

記録を開始したまま席を離れる、PC がスリープする、画面をロックする —— は日常的に起きる。
何も検知しないと「8時間の設計作業」のような記録が残り、統計そのものが信用できなくなる。

## 検知の3経路

無操作の監視だけでは足りない。それぞれ別の仕組みが要る。

| 経路 | 仕組み | なぜ必要か |
|---|---|---|
| 無操作 | `GetLastInputInfo` | アプリが非アクティブでも、システム全体の入力を拾える |
| スリープ・休止 | `SystemEvents.PowerModeChanged` | `GetTickCount` はスリープ中に進まないため、無操作の監視では検知できない |
| 画面ロック | `SystemEvents.SessionSwitch` | 無操作でも拾えるが、理由を「ロック」と出し分けたほうが納得感がある |

スリープとロックは実時間（`DateTime.Now`）の差で期間を測る。

### 一周対策

`GetTickCount` は約49.7日で一周する。
差分は符号なしで引く（`unchecked(GetTickCount() - info.dwTime)`）ことで、
一周をまたいでも正しい値になる。

## 動作

### 記録中のみ集める

検知自体は常時行うが、`_awayDuringRecording` に溜めるのは記録中に起きたものだけ。
記録していない時間の離席は関心の対象外。

### 勝手に削らない

離席を検知しても記録は止めず、変更もしない。
記録中は「◯時◯分から操作がありません」というバナーを出すだけ。

記録を停止したときに、検知した離席の一覧を示して
「離席時間を除く」か「そのまま記録」かを選ばせる。

### 「除く」は分割で表現する

合計から時間を引くだけだと、いつ作業していたのかが失われる。
離席をまたぐ記録は複数のアイテムに分割する。

```
記録:  09:00 ────────────────────────────── 15:00
離席:              11:00 ─── 12:30

結果:  09:00 ─ 11:00        12:30 ─ 15:00   （2件）
```

離席が末尾にある場合は自然に1件へ縮む（＝終了時刻の切り詰め）。

記録全体が離席だった場合は何も作らず、通知だけ出す。
ユーザーが「除く」を選んだ結果なので、勝手に残すほうが不自然。

### 取り消しは1件にまとめる

分割で複数アイテムが生まれても、ユーザーから見れば1回の停止操作。
`CompositeEdit` でまとめ、Ctrl+Z 一回で戻せるようにしている。

## 実装上の注意点

### ポーリングの取りこぼし

無操作の監視は5秒間隔のポーリング。
離席から戻った直後にホットキー（Ctrl+Alt+R）で記録を停止すると、
次のポーリングが来る前に停止処理が走り、**最後の離席期間を取りこぼす**。

記録停止の直前に `FlushPendingAway()` を呼び、進行中の離席をその場で確定させる。

### 離席の開始時刻

閾値を超えた瞬間から遡って「今 − 閾値」とすると、ポーリング間隔ぶんずれる。
実際の無操作時間を使って `DateTime.Now - idle` で求める。

### スリープ復帰直後

復帰直後は無操作時間が大きく出る。
`PowerModes.Resume` で離席状態を仕切り直し、スリープ期間と二重に数えないようにする。

### 重なりの畳み込み

ロックとスリープと無操作は同時に成立しうる。
`SubtractAway` は区間を開始順に並べてから重なりを1つにまとめ、
そのうえで記録区間から差し引く。

## 設定

設定パネルの「一般」タブに追加。

- **離席を検知する** — 既定は有効
- **離席とみなす時間** — 3 / 5 / 10 / 15 / 30 / 60 分（既定10分）

## 構成

| ファイル | 役割 |
|---|---|
| `Models/AwayPeriod.cs` | 離席期間（開始・終了・理由）と表示用の整形 |
| `Services/AwayDetector.cs` | 3経路の監視、期間の確定、`FlushPendingAway` |
| `ViewModels/MainViewModel.Away.cs` | 記録中の収集、`SubtractAway`、区間の保存 |
| `Views/Dialogs/AwayReviewDialog.xaml` | 停止時の確認ダイアログ |
| `Helpers/UndoableEdits.cs` | `CompositeEdit`（分割を1件の履歴にまとめる） |

## 今後の候補

- **無操作が続いたら自動で記録を停止する**オプション。
  「一晩つけっぱなし」を根本的に防げるが、
  現状は「検知して停止時に聞く」だけにしている
- 離席期間を「休憩」カテゴリのアイテムとして残す選択肢
- 離席の一覧から、除外する期間を個別に選ぶ
