# 日/週ビューをタイムラインと揃える

タイムラインを作り込んだ結果、同じアプリ内で日/週ビューとの操作感が食い違っていた。
その差を埋めた記録。

## 訂正: 現在時刻ラインは元からあった

当初「日/週ビューに現在時刻ラインがない」と書いたが、これは誤り。
赤いラインと丸のインジケーターは以前から実装されている。

## 埋めた差

### 1. ツールチップ

15分の予定はバーの高さが約15pxしかなく、タイトルも時刻も潰れて読めなかった。

タイムラインには `MainViewModel.Timeline.cs` の中に `BuildToolTip` を持たせていたが、
同じものを日/週ビューにも書くのは重複になる。
`ScheduleItem.ToolTipText` としてモデル側へ移し、両方から使うようにした。

Title / Content / StartTime / EndTime / IsAllDay の変更で
`ToolTipText` の変更通知も出るよう配線してある。

### 2. 文字色の固定

`Foreground="#1E293B"` / `#64748B` がハードコードされており、
濃いカテゴリ色（DarkOrange など）やダークモードで文字が潰れていた。
タイムラインで直したのと同じ問題。

`BrushToContrastTextConverter` を日/週ビューの通常バーと終日バーにも適用した。
副次テキスト（メモ・時刻）用に `ConverterParameter=Muted` を追加し、
主テキストと同じ側の色を薄くして返す。

### 3. 選択とキーボード操作

**選択はアイテム自身が持つようにした。**
`TimelineBar.IsSelected` を廃止し、`ScheduleItem.IsSelected` へ移した。
ビューごとに別々の選択を持つと、表示を切り替えたときに選択が消えて混乱するため。

これに伴い `SelectedTimelineItem` を `SelectedItem` へ改名し、
選択コマンドも `MainViewModel.Selection.cs` に切り出して共通化した。

| コマンド | 対象 |
|---|---|
| `MoveSelectionCommand` | 表示中のビューに応じて候補が変わる（下記） |
| `EditSelectedCommand` | 選択中のアイテムを編集 |
| `DeleteSelectedCommand` | 選択中のアイテムを削除 |
| `ClearSelectionCommand` | 選択解除 |

選択を動かす候補:

- タイムライン — 仮想化前の全バー（検索で減光中のものは飛ばす）
- 日/週ビュー — 表示中の日に含まれるアイテム（日またぎの重複は除く）

キー割り当ては軸の向きに合わせて変えている。
タイムラインは時間軸が横なので ←→ で選択移動。
日/週ビューは時間軸が縦なので:

| キー | 動作 |
|---|---|
| `↑` `↓` | 前後のアイテムへ選択移動 |
| `←` `→` | 前日 / 翌日へ移動 |
| `Enter` | 編集 |
| `Delete` | 削除 |
| `Esc` | 選択解除 |
| `T` | 今日へ |

### 4. 空き領域の縦ドラッグで新規作成

従来はダブルクリックで固定1時間の予定を作るだけだった。
縦にドラッグすると、その時間帯（15分スナップ）で作成できるようにした。
ダブルクリックの挙動は従来どおり残してある。

週ビューではドラッグ開始時の列で日付を固定する。
縦にドラッグする間に横へ少しぶれても、日が変わらないようにするため。

### 5. ホバー強調

バーにマウスを乗せると不透明度と枠線が上がる。掴める対象だと分かるようにした。

## 実装上の注意点

### DataTemplate の中では x:Name が使えない

日/週ビューの描画面は `TransitioningContentControl` の `DataTemplate` の中にある。
`x:Name` を付けてもコードビハインドのフィールドは生成されないため、
範囲ドラッグのプレビュー矩形は**実行時に生成して描画面へ足す**方式にした。

日付を移動すると描画面ごと作り直されるので、
`EnsureRangePreview` は親が変わっていたら作り直す。

### キーボードハンドラの棲み分け

3つの `PreviewKeyDown` が同時に存在する。トンネリングの順で:

1. `Window_PreviewKeyDownForUndo` — Ctrl+Z / Ctrl+Y のみ。それ以外は素通し
2. `ScheduleView_PreviewKeyDown` — 日/週ビュー。Ctrl 併用時は素通し
3. `TimelineRoot_PreviewKeyDown` — タイムライン

日/週ビューの ScrollViewer は他のビューに覆われても木構造に残るため、
`IsDayMode` / `IsWeekMode` でないときは何もしないようにガードしている。

いずれもフォーカスが `TextBoxBase` にあるときは横取りしない。

## 見送ったもの

**縦ズーム**（`PixelsPerHour` の可変化）。
コンバーター3箇所とドラッグ処理の定数に波及するため、今回の範囲からは外した。
