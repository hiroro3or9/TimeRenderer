# 離席・中断の検知 — 使い方ガイド

## なにをする機能か

記録をつけたまま席を離れてしまった時間を見つけて、記録から差し引けるようにします。

記録を開始したまま昼休みに入る、打ち合わせに呼ばれる、PC がスリープする——
こうしたことは日常的に起こります。放っておくと「8時間の設計作業」のような
実態とかけ離れた記録が残り、集計そのものが当てにならなくなります。

この機能は**勝手に記録を書き換えません**。
離席を見つけて教え、記録を止めたときに「どうするか」を確認するだけです。

## 3種類の中断を見ています

| 種類 | 判定のしかた |
|---|---|
| **無操作** | キーボードとマウスの操作が一定時間なかった |
| **スリープ** | PC がスリープ・休止していた |
| **ロック** | 画面がロックされていた（Win+L、ユーザー切り替えなど） |

一覧にはどの種類だったかが表示されるので、後から見ても状況が分かります。

> **注意：操作の有無は Windows 全体で見ています**
>
> TimeRenderer を開いていなくても、他のアプリで作業していれば「操作あり」です。
> アプリを最小化していても検知は続きます。
>
> 逆に、**キーボードもマウスも触らない作業は「離席」と判定されます**。
> 動画やオンライン講義の視聴、他の人が話している会議、
> 紙の資料を読んでいる時間などが該当します。
> こうした使い方が多い場合は、後述のしきい値を長めにするか、
> 機能をオフにしてください。

## 使っているときの流れ

### 1. 記録中に離席すると、画面上部に帯が出ます

```
⚠ 14:32 から操作がありません。記録は続いています（停止時に除外できます）
```

記録は止まりません。あくまで「気づいていますよ」という通知です。

### 2. 記録を停止すると、確認画面が出ます

離席が1件も検知されなかった場合は、これまでどおり何も出ずに記録されます。

検知されていた場合はこんな画面が出ます。

```
┌──────────────────────────────────────────────┐
│ 「詳細設計」の記録中に離席を検知しました        │
│                                              │
│ 記録全体 09:00 〜 18:00（9時間）のうち、       │
│ 2 件・合計 1時間45分 が離席です。              │
│                                              │
│  ┌────────────────────────────────────────┐  │
│  │ 12:00 〜 13:00 （60分・無操作）          │  │
│  │ 15:20 〜 16:05 （45分・ロック）          │  │
│  └────────────────────────────────────────┘  │
│                                              │
│ 「離席時間を除く」を選ぶと、実作業 7時間15分   │
│ 分のみが記録されます。                        │
│ 離席をまたぐため、記録は複数のアイテムに       │
│ 分割されます。                                │
│                                              │
│              [ そのまま記録 ] [ 離席時間を除く ]│
└──────────────────────────────────────────────┘
```

### 3. どちらかを選びます

**そのまま記録** — 検知は無視して、9:00〜18:00 の1件として記録します。

**離席時間を除く** — 離席していた時間を取り除いて記録します。

## 「離席時間を除く」を選ぶとどうなるか

合計から時間を引くのではなく、**記録を分割**します。
単に短くすると「いつ作業していたのか」が分からなくなるためです。

上の例なら、9時間の記録1件が次の3件になります。

```
元の記録   09:00 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ 18:00

離席              12:00━13:00      15:20━16:05

結果       09:00━12:00   13:00━15:20   16:05━18:00
             3時間        2時間20分      1時間55分
```

### 離席が末尾にあった場合

分割は起きず、終了時刻が早まるだけです。

```
元の記録   09:00 ━━━━━━━━━━━━━━━━━━━━━ 18:00
離席                        16:00━━━━━ 18:00
結果       09:00 ━━━━━━━━━ 16:00
```

「昼に開始して、そのまま夕方まで放置していた」ようなときはこの形になります。

### 記録全体が離席だった場合

記録は作られません。「全体が離席だったため、記録しませんでした」と通知が出ます。

### 選び直したいとき

分割は1回の操作として扱われるので、**Ctrl+Z を一度押せば元に戻ります**。
戻したうえで、必要なら手作業で予定を追加してください。

## 設定

**設定（歯車アイコン）→ 一般タブ → 離席・中断の検知**

| 項目 | 説明 | 初期値 |
|---|---|---|
| 離席を検知する | オフにすると検知も確認画面も出ません | オン |
| 離席とみなす時間 | この時間だけ操作がないと離席と判定します | 10分 |
| 検知したときの扱い | 確認画面を出すか、自動で決めるか | 毎回確認する |

### 「検知したときの扱い」の3つの選択肢

| 選択肢 | 動き |
|---|---|
| **毎回確認する** | 記録を停止するたびに確認画面を出します |
| **常に除外する** | 確認せずに離席分を差し引きます。何を差し引いたかは画面上部に通知され、Ctrl+Z で戻せます |
| **常にそのまま記録する** | 確認画面を出さず、記録も変更しません。記録中の帯（通知）だけが出ます |

離席が毎日ほぼ同じパターンで起きるなら「常に除外する」が楽です。
判断が都度必要なら「毎回確認する」のままにしてください。

### しきい値の決め方の目安

| 設定 | 向いている使い方 |
|---|---|
| 3〜5分 | 短い中断も細かく拾いたい。集計の精度を重視する場合 |
| 10分（初期値） | 一般的な使い方。お茶を淹れる程度では反応しません |
| 15〜30分 | 考えごとや読書、動画視聴など、手を止める時間が長い作業が多い場合 |
| 60分 | 昼休みなど、はっきりした長時間の離席だけ拾いたい場合 |

短くするほど細かく拾えますが、確認画面が出る頻度も上がります。

## こまかい挙動

- **しきい値より短い中断は記録されません。** 10分設定なら、8分席を外しても何も出ません。
- **重なった中断はまとめられます。** 画面をロックしてスリープに入った場合など、
  二重には数えません。
- **記録していない時間の離席は無視します。** 記録中に起きたものだけが対象です。
- **離席から戻ってすぐ記録を開始した場合、その前の離席は含まれません。**
- **記録中でも表示を切り替えて構いません。** 検知は続きます。
- **アプリを終了するときに記録中だった場合**、記録が保存されるため、
  離席があれば確認画面が出ます。

## よくある質問

**Q. 離席と判定されたくない作業があります**

しきい値を長くするか、機能をオフにしてください。
オフにしても、これまでどおり記録・集計は使えます。

**Q. 過去の記録に対して後から離席を差し引けますか**

できません。検知は記録中にのみ行われます。
過去の記録は、予定をダブルクリックして時刻を直接編集してください。

**Q. 確認画面で間違えて「そのまま記録」を選びました**

記録自体はやり直せませんが、Ctrl+Z で追加を取り消し、
記録を作り直すか、時刻を手で編集できます。

**Q. 検知は動いているのに帯が出ません**

帯は**記録中の離席**でのみ表示されます。
記録していないときの離席では何も出ません。

**Q. スリープから復帰したら、スリープ中の時間が丸ごと離席になっていました**

仕様どおりです。スリープ中は PC が停止しているため、
その時間は作業していないものとして扱います。
